# 知识库写作规则
## 目录结构

- `inputs/`：用户放入原始资料的地方（文章、笔记、摘录等）
- `knowledge_base/index.md`：全局索引，列出所有知识库名称和简介
- `knowledge_base/<库名>/index.md`：该库的局部索引，说明库的主题和文件列表
- `knowledge_base/<库名>/<文章>.md`：知识库中的具体文章

## 文章写作

- 每个知识库的所有文章共同构成一个结构化、可索引的知识网络；写作时注重提炼关键原则和观点，并配以具体例子，使知识既便于检索，也便于理解和推理
- 每篇文章聚焦一个概念或主题，内容来自对原始资料的提炼和综合，而非简单复制
- 文章之间用 `[[文章名]]` 或 Markdown 链接互相引用（backlink）
- 可引用图片：使用 `![描述](相对路径)` 语法（如 `![架构图](images/arch.png)`），图片文件存放在文章同目录下（如 `images/` 子目录），支持 png/jpg/gif/webp 格式
- [注意]提炼或总结的内容要精炼，不要啰嗦

## 索引维护

- 新增文章后，同步更新该库的 `index.md` 文件列表
- 新增知识库后，同步更新全局 `knowledge_base/index.md`
- 全局索引表格格式为 `| 知识库名 | 简介 |`
- 删除文章或知识库后，同步清理对应的索引条目

## 整理原始资料流程
1. 读取 `inputs/` 中的文件，理解内容
2. 判断归属：放入已有知识库，或新建知识库；不确定时询问用户
3. 对于每一篇原始资料或一类原始资料，与用户确认提炼和总结的重点与方向，待用户确认后再进行写作
4. 将内容提炼为文章，写入对应知识库目录
5. 更新索引
